# Tram Predictive

Проект по созданию предиктивной аналитики для трамвайного транспорта.

Основная задача — анализировать данные о работе трамваев и прогнозировать возможные неисправности, в первую очередь ошибки и отказы дверей.

---

# Назначение репозитория

Репозиторий содержит:

- описание бизнес-задачи;
- источники и структуру данных;
- архитектуру решения;
- модель данных;
- предиктивную модель;
- аналитические расчёты;
- визуализацию результатов;
- план развития проекта.

# Основные принципы

- Сначала понять бизнес-задачу и данные, затем выбирать алгоритм.
- Предиктивная модель создаётся только после проверки качества данных.
- Целевое событие должно иметь точное и понятное определение.
- Результат модели должен быть полезен для принятия решений.
- Простое и объяснимое решение предпочтительнее неоправданно сложного.
- Документация создаётся по мере развития проекта.
- Каждый документ отвечает только на один вопрос.
- Не создавать лишние документы заранее.

# Ближайший этап

1. Описать бизнес-задачу.
2. Понять, какие ошибки дверей фиксируются.
3. Получить примеры исходных данных.
4. Определить, что именно должна прогнозировать модель.
5. Оценить, достаточно ли данных для построения прогноза.
